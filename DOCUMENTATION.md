## 1. نظرة عامة (غير تقنية)

**Teacher Hub** تطبيق سطح مكتب لنظام Windows موجّه للمعلّم الذي يُدرّس أونلاين بشكل
فردي. يحلّ مشكلة تتبّع الحصص والمدفوعات يدوياً عبر دفاتر أو جداول Excel متناثرة، ويجمع كل
ما يحتاجه المعلّم في مكان واحد بواجهة عربية كاملة من اليمين لليسار (RTL):

- من **هو الطالب** وبياناته (اسم، أرقام تواصل، رابط Zoom، أيام وأوقات الحصص).
- **كم حصة** أخذها الطالب في الدورة الحالية، ومتى يستحق الدفع.
- توليد **فاتورة PDF احترافية بالعربية** وإرسالها عبر واتساب.
- **تذكيرات** بالحصص القادمة عبر منطقة الإشعارات (System Tray).
- **تقارير شهرية** بالدخل وعدد الحصص وأكثر الطلاب نشاطاً.

الفلسفة التشغيلية: **محلّي أولاً (offline-first)** — كل البيانات في ملف قاعدة بيانات
واحد على جهاز المعلّم (`data/teacher.db`)، بلا خادم ولا حساب ولا إنترنت إلزامي (الإنترنت
يُستخدم فقط لفتح Zoom/واتساب).

النموذج المالي المحوري: **دورة من 8 حصص** — عند إكمال الطالب 8 حصص محتسَبة يظهر كـ
"مستحق للدفع"، ويولّد المعلّم فاتورته ثم يصفّر العدّاد ليبدأ دورة جديدة.

## 4. المعمارية العامة

التطبيق مبني بنمط **طبقي صارم (Layered)** يفصل المنطق عن الواجهة:

```
main.py  (نقطة الدخول، مُسجّل أعطال، قفل نسخة واحدة، خطوط، ثيم)
   │
   ▼
app/ui/        ← الواجهة فقط (PySide6): نوافذ، صفحات، ودجتس، مساعدات
   │  تستدعي ▼
app/services/  ← منطق العمل: طلاب، حصص، فواتير، PDF، Excel، إشعارات، روابط
   │  تستدعي ▼
app/db/        ← النماذج (ORM) + المحرّك (engine/session)
   │
   ▼
app/config.py  ← الثوابت والمسارات (يعتمد عليها الجميع)
```

**مبادئ حاكمة:**
- الواجهة **لا تلمس** قاعدة البيانات مباشرة؛ تمرّ دائماً عبر `app/services`.
- الخدمات **لا تعرف** شيئاً عن Qt (قابلة للاختبار بمعزل عن الواجهة) — باستثناء
  `launcher.py` و`notification_service.py` اللذين يستخدمان Qt لأسباب وظيفية (فتح روابط،
  مؤقّت).
- التواصل بين الصفحات يتم عبر إشارات Qt (`Signal`)؛ خصوصاً إشارة `data_changed`.

## 6. نموذج البيانات (Schema)

قاعدة البيانات SQLite عبر SQLAlchemy ORM (`app/db/models.py`). ستة جداول:

### 6.1 `Student` (الطلاب)
| الحقل | النوع | ملاحظات |
|------|------|---------|
| `id` | Integer PK | المعرّف |
| `name` | String, NOT NULL | الاسم (مطلوب، يُتحقّق منه) |
| `phone` | String | رقم الطالب/واتساب |
| `parent_phone` | String | رقم ولي الأمر |
| `zoom_link` | String | رابط Zoom |
| `zoom_link_name` | String | اسم/وصف رابط Zoom |
| `whatsapp_group_link` | String | رابط مجموعة واتساب |
| `price_per_session` | Float, NOT NULL=0 | سعر الحصة (≥ 0) |
| `sessions_per_cycle` | Integer, NOT NULL=8 | عدد حصص الدورة (≥ 1) |
| `weekly_schedule` | String (JSON list) | أيام الأسبوع (موروث) |
| `session_time` | String | الوقت الموروث (legacy) |
| `day_schedules` | String (JSON `{day:[times]}`) | الجدول الحديث: أوقات متعددة لكل يوم |
| `custom_fields` | Text (JSON list) | حقول يعرّفها المستخدم `{label,value,show_in_popup}` |
| `notes` | Text | ملاحظات حرّة |
| `created_at` | DateTime = `now` | تاريخ الإنشاء |
| `is_active` | Boolean = True | للحذف الناعم (soft delete) |

علاقات: `sessions`, `videos`, `invoices` — جميعها `cascade="all, delete-orphan"`
(حذف الطالب نهائياً يحذف أبناءه).

### 6.2 `Session` (الحصص)
`id`, `student_id` (FK), `session_date` (DateTime=now), `cycle_number` (Int=1),
`counted` (Bool=True — هل تُحتسب في الدورة الحالية)، `is_free` (Bool=False — حصة

### 6.3 `Video` (الفيديوهات/التسجيلات المرسلة)
`id`, `student_id` (FK), `sent_date` (DateTime=now), `description` (String),
`counted` (Bool=True).

### 6.4 `Invoice` (الفواتير)
`id`, `student_id` (FK), `issued_at` (DateTime=now), `sessions_count` (Int),
`videos_count` (Int), `amount` (Float), `pdf_path` (String), `notes` (Text),
`is_paid` (Bool=False), `paid_at` (DateTime, nullable).

### 6.5 `Setting` (الإعدادات)
متجر مفتاح/قيمة: `key` (String PK), `value` (Text — قيمة JSON مُسلسَلة).

### 6.6 `WhatsAppGroup` (مجموعات واتساب)
`id`, `name` (String 120, NOT NULL), `invite_link` (String 255, NOT NULL),
`notes` (Text), `created_at` (DateTime=now).

## 7. طبقة قاعدة البيانات

ملف `app/db/engine.py`:

- **المحرّك**: `create_engine(DB_URL, future=True)` حيث `DB_URL = sqlite:///data/teacher.db`.
- **الجلسات**: `scoped_session(sessionmaker(autoflush=False, expire_on_commit=False))`.
  - `scoped_session` = جلسة واحدة لكل خيط (thread-local)، يعاد استخدامها عبر `get_session()`.
  - `expire_on_commit=False` يبقي الكائنات قابلة للقراءة بعد `commit` (مهم لتمريرها للواجهة).
- **هجرات خفيفة** (`_apply_column_migrations`): عند الإقلاع تُضاف الأعمدة الناقصة عبر
  `ALTER TABLE ... ADD COLUMN` لجداول موجودة (تطوّر تدريجي للسكيمة دون أدوات هجرة ثقيلة).
  القائمة `_MIGRATIONS` تشمل أعمدة أُضيفت لاحقاً (مثل `is_free`, `paid_at`, `day_schedules`,
  `custom_fields`، إلخ).
- **`init_db()`**: ينشئ الجداول (`create_all`) ثم يطبّق الهجرات. يُستدعى من `main.py`.
- **`get_session()`**: يعيد الجلسة الحالية (المشتركة).
- **`configure_engine(db_url)`**: يعيد ربط المحرّك والجلسات بقاعدة أخرى — تستخدمه
  الاختبارات لتوجيه كل الوصول لقاعدة مؤقتة معزولة عن القاعدة الحقيقية.
- **`session_scope()`** (context manager): commit عند النجاح وrollback عند الخطأ —
  الأداة المعتمدة لعمليات الكتابة الآمنة.
- دعم متغيّر البيئة `TEACHER_DB_URL` لإعادة توجيه القاعدة دون تعديل الكود.

## 8. طبقة الخدمات (Business Logic)

### 8.1 `student_service.py`
- `list_students(active_only=True)` — قائمة الطلاب مرتّبة بالاسم.
- `get_student(id)` — جلب طالب.
- `create_student(...)` — إنشاء **مع تحقّق**: الاسم غير فارغ، السعر ≥ 0،
  `sessions_per_cycle` ≥ 1 (وإلا `ValueError`). يشتقّ `weekly_schedule`/`session_time`
  من `day_schedules`، ويُسلسل الحقول لـ JSON.
- `update_student(id, **fields)` — تحديث ديناميكي مع نفس قواعد التحقّق.
- `delete_student(id, soft=True)` — حذف ناعم (تعطيل) أو نهائي.
- مساعدات JSON آمنة (تعيد قيماً فارغة عند التلف دون انهيار):
  `get_weekly_schedule`, `get_custom_fields`, `get_day_schedules` (تطبّع الصيغ
  القديمة/الجديدة)، `_normalize_schedules`, `_sanitize_custom_fields`.
- **العدّ**: `counted_sessions(id)` (محتسبة وغير مجانية)، `total_counted_sessions(id)`
  (تشمل المجانية)، `counted_videos(id)`.
- **عدّ مُجمّع (للأداء)**: `counted_sessions_map()` و`counted_videos_map()` —
  يعيدان `{student_id: count}` باستعلام تجميع واحد (`GROUP BY`).

### 8.2 `session_service.py`
`add_session`, `add_video`, `undo_last_session` (يحذف آخر حصة محتسبة ويعيد لقطة dict)،
`update_session`, `list_sessions`, `list_videos`, `delete_session`, `delete_video`.

### 8.3 `billing_service.py`
- `students_with_dues()` — يستخدم خرائط العدّ المُجمّعة؛ يُرجع من بلغوا حدّ الدورة مع
  `amount = round(count × price, 2)` (تقريب نقدي لخانتين). يتجاهل `cycle ≤ 0`.
- `reset_cycle(id)` — يجعل حصص/فيديوهات الطالب `counted=False` لبدء دورة جديدة.
- `record_invoice(...)`, `list_invoices(only_paid=None)`, `mark_invoice_paid(id, paid)`,
  `delete_invoice(id)`.
- `backup_database()` — نسخة احتياطية مؤرّخة في `data/backups/`.
- `reset_all_activity(keep_students=True)` — **تصفير شامل** بعد أخذ نسخة احتياطية: يحذف
  كل الحصص/الفيديوهات/الفواتير (واختيارياً الطلاب)، ويعيد عدّادات ما حُذف.
- `monthly_stats(year, month)` — دخل/محصّل/معلّق/عدد حصص/أكثر 5 طلاب عبر تجميعات SQL.

### 8.4 `settings_service.py`
متجر key/value يُسلسِل القيم JSON. واجهات مريحة: `get_theme/set_theme`,
`get_templates/save_templates` (قوالب واتساب الافتراضية مضمّنة)،
`get/set_notifications_enabled`, `get/set_notification_minutes`,
`get/set_invoice_message_template`. يتعامل بهدوء مع القيم التالفة وأخطاء القاعدة.

### 8.5 `group_service.py`
`validate_invite_link` (regex: `^https://chat\.whatsapp\.com/[A-Za-z0-9]+/?$`) + CRUD
لمجموعات واتساب.

## 10. قواعد العمل الأساسية

1. **الدورة = 8 حصص** (`DEFAULT_SESSIONS_PER_CYCLE = 8`، قابلة للتخصيص لكل طالب ≥ 1).
2. **الحصة المحتسَبة**: `counted=True AND is_free=False`. الحصص المجانية/التجريبية
   (`is_free=True`) **لا** تُحتسب في حدّ الاستحقاق ولا في المبلغ.
3. **الاستحقاق**: يظهر الطالب كـ"مستحق" حين `counted_sessions ≥ sessions_per_cycle`
   (بشرط `sessions_per_cycle > 0`).
4. **المبلغ** = `عدد الحصص المحتسبة × سعر الحصة`، **مقرّباً لخانتين** عشريتين.
5. **التصفير بعد التحصيل**: `reset_cycle` يجعل الحصص/الفيديوهات `counted=False` (تبقى في
   السجلّ التاريخي لكنها تخرج من الدورة الحالية).
6. **التصفير الشامل**: `reset_all_activity` يأخذ نسخة احتياطية أولاً ثم يمسح النشاط —
   لا شيء يُفقد نهائياً.
7. **الحذف الناعم**: حذف الطالب افتراضياً = `is_active=False` (يختفي من القوائم النشطة
   ويبقى في التصدير الكامل والتقارير التاريخية).
8. **العملة**: `ج.م` (جنيه مصري) — `CURRENCY` في `config.py`.

## 11. دعم العربية و RTL وتوليد PDF

- التطبيق كلّه RTL: `app.setLayoutDirection(Qt.RightToLeft)`.
- **الخطوط**: عند الإقلاع يحمّل `main.py` خطوط Cairo (Regular/Medium/SemiBold/Bold) من
  `app/resources/fonts/`، ويسقط إلى Tajawal/Amiri ثم Segoe UI إن غابت.
- **توليد PDF** (`pdf_service.py`):
  - تسجيل خط بأولوية: **Tahoma → Arial** (خطوط نظام بتغطية عربية كاملة) → **Cairo**
    المرفقة → fallback إلى Helvetica.
  - **`ar(text)`**: يعيد تشكيل الحروف عبر `ArabicReshaper` (بلا ligatures لتفادي محارف
    ناقصة) ثم يطبّق اتجاه RTL عبر `python-bidi` (`get_display(..., base_dir="R")`).
  - **`_safe_filename`**: يبني اسم ملف آمن — يُبقي الأحرف الأبجدية الرقمية والعربية
    (`ord > 127`) ويزيل الفواصل و`..`/`/` (يمنع الخروج من المجلد).
  - تصميم الفاتورة: عنوان، رقم/تاريخ، جدول بيانات الطالب، جدول الحصص، جدول الفيديوهات
    (إن وُجدت)، ملاحظات، صندوق الإجمالي المُبرَز، وتذييل شكر. تُحفظ في `exports/invoices/`
    باسم `{اسم الطالب}_{التاريخ_الوقت}.pdf`.

## 12. التكاملات الخارجية (واتساب/Zoom)

- **Zoom**: `open_zoom(link)` يفتح الرابط عبر متصفّح/تطبيق Zoom.
- **واتساب (فردي)**: `open_whatsapp(phone, message)` ينظّف الرقم (يزيل غير الأرقام)،
  يجرّب نظام `whatsapp://send?phone=...&text=...`، وإلا يسقط إلى `https://wa.me/...`.
- **واتساب (مجموعة)**: لا تدعم روابط الإرسال المباشر للمجموعات، لذا يُنسخ نص القالب
  إلى الحافظة ثم يُفتح رابط الدعوة ليُلصِق المعلّم الرسالة يدوياً (Ctrl+V).
- **قوالب الرسائل**: نصوص فيها متغيّرات `{name}`, `{time}`, `{zoom}`, `{sessions}`,
  `{amount}`, `{videos}`, `{date}` تُستبدل عبر `render_template`.
- **إرفاق PDF**: لا يدعم سكيم واتساب الإرفاق برمجياً؛ يُرشَد المعلّم لسحب ملف الـ PDF
  وإفلاته في المحادثة.

## 15. تحسينات الأداء

طُبِّقت أربع طبقات تحسين تحافظ على السلوك (نفس الأرقام والنتائج):

1. **القضاء على N+1**: استبدال عدّ لكل طالب بـ `counted_sessions_map()` /
   `counted_videos_map()` (استعلامَا تجميع `GROUP BY`)، مُستخدَمة في `students_with_dues`،
   صفحة الطلاب، جدول اليوم، وتصدير Excel → من `O(N)` استعلام إلى عدد ثابت.
2. **التحديث الكسول**: مجموعة `_dirty` في النافذة الرئيسية تحدّث الصفحة المرئية فقط عند
   التغيير، وتؤجّل الباقي حتى الانتقال إليها.
3. **معالجة I/O بالخلفية**: توليد PDF وتصدير Excel عبر `run_in_background` (QThreadPool)
   لمنع تجميد الواجهة، مع تنظيف جلسة الخيط وإعادة النتائج عبر إشارات.
4. **تحسينات دقيقة**: `lru_cache` للأيقونات، وdebounce (80ms) لإعادة ترتيب شبكة البطاقات
   أثناء تغيير حجم النافذة.

## 20. قيود معروفة وملاحظات

- **محلّي فقط**: لا مزامنة سحابية ولا تعدّد مستخدمين؛ البيانات على جهاز واحد (انسخ
  `data/` احتياطياً يدوياً للأمان، إضافةً للنسخ التلقائية قبل التصفير الشامل).
- **واتساب**: لا إرفاق PDF برمجياً، والمجموعات تتطلّب لصقاً يدوياً (قيد منصّة واتساب).
- **أخطاء mypy المتبقّية إيجابيات كاذبة**: ناتجة عن عدم استخدام إضافة SQLAlchemy لـ mypy
  (مثل `Base is not valid as a type` و`Column[...]`)، وليست أخطاء وقت تشغيل.
- **التوقيت محلّي (naive)**: كل التواريخ بتوقيت الجهاز المحلّي (`datetime.now`) — مناسب
  لتطبيق أحادي الجهاز.
- **التصدير/الاسترجاع**: لا توجد واجهة لاستيراد نسخة احتياطية؛ الاسترجاع يدوي باستبدال
  ملف القاعدة من `data/backups/`.
